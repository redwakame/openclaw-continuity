# Frontstage notes

This file stays intentionally small. V2 does not own persona design; it protects continuity-facing frontstage behavior and keeps internal reasoning out of user-visible text.

## V2 frontstage rules

- keep frontstage text in natural Traditional Chinese
- avoid exposing hook, schedule, checker, runtime, or internal-source language to the user
- avoid unresolved fragments that require the user to guess what `這個` or `that` refers to
- avoid false closure when no concrete continuity anchor exists
- apply sleep/rest suppress only to follow-up behavior, not to ordinary user-facing replies

## V2 follow-up tone boundaries

- `emotional_followup`: accompany first, ask lightly
- `health_followup`: check condition gently, do not sound like generic health advice
- `progress_followup`: continue a parked task or topic without sounding like a manager reminder
- `tomorrow_check`: light next-step reminder with human warmth

## Optional note

Rhythm or nudge variants belong to optional layers. They are not part of the public V2 default unless a host explicitly enables and supports them.
